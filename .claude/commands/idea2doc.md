## 步骤

**第一步：读取输入**

读取简报文件：`$ARGUMENTS`

**第二步：确定输出路径**

例：`quarterly.idea.md` → `handoff/quarterly.qpad.md`

**第三步：生成 .qpad.md**

按以下规则生成完整文档并写入输出路径。

> 若简报中缺少必要 META 字段（文档类型、输出格式、语气基调、规格约束），停止并列出缺失项。

### YAML 前言区规则

```yaml
---
quillen_pad:
  version: 1
  kind: <report|memo>
  title: "<标题>"
  lang: zh-CN
template: report        # 输出格式=Word 时加；Markdown/PPT 时省略此行（TODO：非 Word 输出路径后续版本支持）
layout:
  list_left: 0
  list_hanging: 0
---
```

字段映射：
- `kind`：简报 META.文档类型 = 报告或技术文档 → `report`；其他 → `memo`
- `title`：从简报「核心目的」中提炼简洁标题，不超过 20 字
- `template`：简报 META.输出格式 = Word → 加 `template: report`；否则省略整行
- `layout`：固定填 `list_left: 0 / list_hanging: 0`（后续版本扩展）

### 正文写作规则

**字数约束**：总字数在简报「规格约束.总字数」的 ±10% 内；按「章节分配」比例分配各章字数，注意代码块不计入字数
**禁区执行**：「禁区」中每一条逐一遵守；禁区内容不出现在正文，内化为写作约束执行，不做任何标注
**语气基调**：按简报 META.语气基调（正式/半正式/技术中性）行文
**素材使用**：【主干】素材全部体现；【支撑】按需；【背景】仅在有助理解时引入

### 写文件规则

- 文件以 `---` 开头，不加任何前后说明文字
- **标题不写手写序号**（"一、""1.1""Chapter 1"等全部禁止）；序号由排版程序自动注入
- **正文顶部不写总标题**；文档标题只在 front matter `title` 字段，写了会出现双标题
- **块级公式 `$$` 前后各留一个空行**，否则排版程序解析失败
- **表格列数必须与表头一致**，否则排版程序报错
- 正文用标准 Markdown；行内公式 `$...$`、块级公式 `$$...$$`；表格用 GFM pipe 格式

**第四步：字数质检 → 扩写循环**

文件写入后，运行质检：

```bash
conda run -n quillen python desk/check.py <输出路径> --target <总字数范围>
```

- `--target` 从简报「规格约束.总字数」中读取（如 `7500-8500`）
- 若质检**通过**（exit 0），进入第五步
- 若质检**未通过**（exit 1），执行以下扩写规则后重写文件，再次运行质检，**最多扩写两轮**：
  - 两轮内通过 → 进入第五步
  - 两轮后仍未通过 → 报告用户当前字数与缺口，然后**直接继续执行第五步**，不再扩写

### 扩写规则（每轮执行）

1. **优先扩展有参考资料支撑的段落**：简报「关键素材」中标注【主干】的内容，尽可能充分展开，字数越多越好
2. **按缺口最大的章节优先补写**：根据简报各章字数分配，哪章最远离目标就优先扩展那章
3. **允许扩展的方向**：
   - 增加原理解释、背景说明、设计动机
   - 细化参数含义、列举更多示例
   - 补充使用场景、注意事项、边界条件
4. **禁止为凑字数做的事**：引入禁区限制的内容

**第五步：调用 desk 生成产物**

质检通过后，运行：

```bash
conda run -n quillen python desk/compose.py <输出路径>
```

输出 DOCX 到默认的 `outbox/` 目录。
